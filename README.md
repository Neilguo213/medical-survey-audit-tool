# 病例问卷审核工具

一个本地网页小工具，用于读取 `doctor.wenwo.com/medic/nsurvey/...` 问卷答卷链接，或粘贴问卷文本，并根据 CRF 字段配置和质控规则生成结构化审核意见。

工具定位是 CRF 质控和逻辑复核，不用于临床诊疗决策。无法确定的问题会输出“需人工复核”“建议补充”或“识别置信度较低，请确认”。

## 启动方式

需要安装 Node.js 18 或更高版本。

macOS 推荐双击：

```text
启动审核工具.command
```

Windows 推荐双击：

```text
start-windows.bat
```

命令行方式：

```bash
npm start
```

然后打开：

```text
http://localhost:4173
```

## 审核流程

1. 粘贴问卷链接并点击“读取链接”，或直接粘贴问卷页面文本。
2. 选择模板：乳腺癌 CRF 或血液肿瘤 CRF。
3. 选择审核严格度。
4. 检查“识别结果”和“未匹配内容”，低置信度字段可人工确认。
5. 点击“开始审核”生成审核意见。
6. 复制报告，或导出 Markdown / JSON。

## 模板说明

当前只保留两个固定模板：

- 乳腺癌 CRF
- 血液肿瘤 CRF

模板字段位于：

- `src/templates/breastCancerTemplate.js`
- `src/templates/hematologyTemplate.js`

每个字段包含稳定字段 ID、中文字段名、模块、别名、字段类型、必填级别、适用条件、依赖字段、选项、单位和说明等信息。字段匹配不依赖固定题号，会优先按题干和 aliases 匹配。

## 审核严格度

标准模式：

- 输出高风险、关键逻辑冲突和关键必填缺失。
- 减少 recommended 和 optional 字段缺失提示。
- 更适合日常快速审核。

严格模式：

- 输出 required + recommended 字段完整性提示。
- 增加边界异常、选项一致性和治疗记录完整性提示。
- 更适合正式质控前复核。

## 主要规则

通用规则包括：

- 身高、体重合理范围复核。
- BMI、BSA 与身高体重的一致性校验。
- ECOG 范围校验。
- 日期格式和治疗、疗效评价、随访时间线校验。

乳腺癌规则包括：

- 分子分型选项检查。
- HER2+ 时严格模式关注 HER2 靶向治疗记录。
- HR+ 时严格模式关注内分泌治疗记录。
- IV 期关注转移信息。
- RECIST 与肿瘤最大径变化的一致性。
- 因副作用停药与不良反应记录一致性。

血液肿瘤规则包括：

- 疾病类型选项检查。
- MM / 多发性骨髓瘤关注 ISS 分期。
- 淋巴瘤严格模式关注 IPI/R-IPI。
- CML 严格模式关注 BCR-ABL1。
- CAR-T 严格模式关注 CRS/ICANS 记录完整性。
- 移植病例关注移植类型，严格模式关注 GVHD 记录。
- 疾病类型与疗效评价字段一致性。

## 导出功能

支持导出：

- Markdown：适合人工阅读、粘贴到文档或沟通工具。
- JSON：适合后续系统处理或留存。

导出内容包括审核时间、模板名称、审核严格度、字段识别结果、未匹配题目、审核意见和风险统计。

## 测试

运行基础测试：

```bash
npm test
```

测试覆盖：

- 文本解析：题干答案同行、分行、多选、未匹配内容。
- 字段匹配：aliases、模糊匹配、低置信度。
- 计算：BMI、BSA。
- 审核引擎：HER2+、MM、淋巴瘤、RECIST 逻辑冲突、动态跳题适用条件。

## 当前限制

- 链接读取依赖 wenwo 当前答卷接口结构，接口变更、权限变化或链接过期可能导致读取失败。
- 文本解析是质控辅助解析，复杂表格、图片、特殊组件仍可能需要人工确认。
- 未匹配内容支持人工映射到字段，但映射结果只保留在当前页面状态中。
- 医学规则保持保守，只做 CRF 完整性、格式、范围和逻辑复核，不输出绝对化临床判断。
